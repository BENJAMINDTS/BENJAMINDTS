## 📐 My Dev Principles

<table>
  <tr>
    <td width="170">🧼 &nbsp;<b>Clean Code</b></td>
    <td>Funciones &lt; 30 líneas y nombres que no mientan.</td>
  </tr>
  <tr>
    <td>📖 &nbsp;<b>Documentation</b></td>
    <td>JSDoc / PHPDoc / JavaDoc — siempre, sin excepciones.</td>
  </tr>
  <tr>
    <td>🏛️ &nbsp;<b>Architecture</b></td>
    <td>SOLID + separación de capas (MVC / Clean Architecture).</td>
  </tr>
  <tr>
    <td>🗃️ &nbsp;<b>Database</b></td>
    <td><code>snake_case</code>, soft delete y migraciones versionadas.</td>
  </tr>
  <tr>
    <td>🌐 &nbsp;<b>API</b></td>
    <td>REST + respuestas JSON estandarizadas + versionado <code>/v1/</code>.</td>
  </tr>
  <tr>
    <td>🔒 &nbsp;<b>Security</b></td>
    <td>Nunca secretos en el repo. Validar absolutamente todo input.</td>
  </tr>
  <tr>
    <td>🌿 &nbsp;<b>Git</b></td>
    <td>Conventional Commits · ramas <code>feat/ fix/ refactor/ docs/</code></td>
  </tr>
  <tr>
    <td>🧪 &nbsp;<b>Testing</b></td>
    <td>Si es lógica de negocio crítica, está testeada.</td>
  </tr>
</table>
